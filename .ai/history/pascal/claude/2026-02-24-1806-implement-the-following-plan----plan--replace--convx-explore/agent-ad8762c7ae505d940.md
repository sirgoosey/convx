### 5. `src/convx_ai/engine.py` (Full Contents)

**File: `/Users/pascal/Code/business/convx/src/convx_ai/engine.py` (233 lines)**

Core orchestration engine for idempotent session synchronization.

**Key classes:**

**SyncResult dataclass:**
```python
@dataclass
class SyncResult:
    discovered: int = 0   # Total files found
    exported: int = 0     # New sessions exported
    skipped: int = 0      # Unchanged (fingerprint match)
    filtered: int = 0     # Invalid/filtered out
    updated: int = 0      # Updated sessions
    dry_run: bool = False
```

**Key functions:**

- `_load_index(index_path)` — Loads `.convx/index.json` or returns empty
- `_relative_cwd_parts(cwd)` — Strips home directory from cwd for nested output
- `_build_output_dir(base_dir, session, flat)` — Builds output path (flat or nested)
- `_session_basename(session, preferred)` — Generates `YYYY-MM-DD-HHMM-slug.md` filename
- `_session_contains(session, needle)` — Recursively searches for skip pattern
- `_is_under_repo(cwd, repo_path)` — Checks if session's cwd is under filter repo
- `sync_sessions()` — Main orchestrator

**Idempotency logic:**
- Fingerprints source files (SHA-256)
- Checks `.convx/index.json` for cached fingerprints
- Skips if fingerprint unchanged AND both markdown and JSON files exist
- Updates index on each sync
- Creates `.convx/.gitignore` to hide index internals

**Output handling:**
- Creates two files per session:
  - Markdown: `YYYY-MM-DD-HHMM-slug.md` (readable transcript)
  - JSON: `.YYYY-MM-DD-HHMM-slug.json` (normalized data)
- Supports child sessions (agent threads) with separate `.md` files
- Atomic writes to prevent corruption
